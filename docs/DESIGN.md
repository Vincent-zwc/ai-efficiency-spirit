### 1.2 架构分层

| 层级 | 职责 | 技术实现 |
|------|------|---------|
| **展示层 (UI)** | 页面渲染、用户交互、动画效果 | React 19 + shadcn/ui + Tailwind CSS 4 |
| **状态层 (State)** | 客户端状态管理、数据持久化 | Zustand 5 + persist middleware |
| **通信层 (Network)** | 前后端数据传输 | fetch API + HTTP POST |
| **服务层 (Server)** | AI 调用、数据预处理 | Next.js API Routes + z-ai-web-dev-sdk |
| **数据层 (Data)** | 结构化数据存储 | localStorage + Prisma/SQLite |

### 1.3 设计原则

1. **单一职责**：每个模块组件只负责一个功能领域
2. **关注点分离**：AI 调用仅在后端执行，前端只负责展示和交互
3. **统一接口**：所有 AI 调用通过 `/api/ai` 统一路由，差异通过 systemPrompt 区分
4. **状态集中**：全局状态集中在 Zustand Store，避免 prop drilling
5. **渐进增强**：AI 功能失败时提供 fallback 方案（如百词斩内置词库）

## 2. 模块设计

### 2.1 页面路由与导航

由于赛事环境限制（用户只能看到 `/` 路由），采用**单页应用 + 客户端路由**模式：

```
/ (page.tsx)
├── 侧边栏 (Sidebar)
│   ├── 工作台首页 (Dashboard)
│   ├── 智能文档助手 (DocumentAssistant)
│   ├── 会议智能助手 (MeetingAssistant)
│   ├── 智能待办管家 (TodoManager)
│   ├── 数据报表助手 (DataReport)
│   └── 创意工坊 (CreativeWorkshop)
└── 内容区 (Main Content)
    └── 根据 activeModule 渲染对应模块
```

**路由切换机制**：
- 使用 Zustand Store 的 `activeModule` 字段控制当前展示模块
- 侧边栏点击 → `setActiveModule()` → 组件重新渲染
- URL 不变，通过 `key={activeModule}` 触发切换动画

**AI 智能排序流程**：
```
1. 收集所有任务标题 → "任务1\n任务2\n任务3..."
2. 发送给 AI（systemPrompt: 任务管理助手，返回 JSON 格式）
3. AI 返回 → [{"title":"任务1","priority":"高","suggestedTime":"今天"}, ...]
4. 解析 JSON，匹配并更新每个任务的 priority
5. 按优先级排序：高(0) > 中(1) > 低(2)
6. replaceTodos(sortedTodos) 更新 Store
```

**AI 任务分解流程**：
```
1. 用户点击 [🌳] 按钮
2. 打开 Dialog 弹窗
3. 将任务标题+描述发送给 AI
4. AI 返回子任务列表（每行一个）
5. 解析为 subtasks 数组 → setTodoSubtasks(id, subtasks)
6. 在任务卡片中以缩进形式展示子任务
```

## 3. 状态管理设计

### 3.1 Store 数据模型

```typescript
interface AppState {
  // 导航状态
  activeModule: 'dashboard' | 'document' | 'meeting' | 'todo' | 'data' | 'creative';

  // 待办事项
  todos: TodoItem[];
  todoFilter: 'all' | 'active' | 'completed';

  // 抽奖系统
  lotteryEntries: LotteryEntry[];
  lotteryHistory: LotteryHistory[];

  // 百词斩
  wordQuestions: WordQuestion[];
  currentQuestionIndex: number;
  wordScore: number;
  wordStreak: number;
  wrongWords: WordQuestion[];
}
```

### 3.2 核心数据类型

```typescript
interface TodoItem {
  id: string;              // crypto.randomUUID()
  title: string;           // 任务标题
  description: string;     // 任务描述
  completed: boolean;      // 完成状态
  priority: '高' | '中' | '低';  // 优先级
  createdAt: number;       // 创建时间戳
  subtasks?: string[];     // AI 分解的子任务
}

interface LotteryEntry {
  id: string;
  name: string;
}

interface LotteryHistory {
  id: string;
  winner: string;
  participants: string[];
  timestamp: number;
}

interface WordQuestion {
  word: string;            // 英语单词
  meaning: string;         // 中文释义
  options: string[];       // 4个选项
  answer: number;          // 正确选项索引 (0-3)
}
```

### 3.3 持久化策略

使用 Zustand 的 `persist` 中间件，选择性地持久化以下字段到 localStorage：

```typescript
partialize: (state) => ({
  todos: state.todos,
  todoFilter: state.todoFilter,
  lotteryEntries: state.lotteryEntries,
  lotteryHistory: state.lotteryHistory,
  wordScore: state.wordScore,
  wrongWords: state.wrongWords,
})
```

**不持久化的字段**：
- `activeModule`：每次打开默认进入首页
- `wordQuestions`：每次需要重新生成
- `currentQuestionIndex`、`wordStreak`：答题会话数据，不跨会话保留

## 4. API 设计

### 4.1 AI 统一接口

**端点**：`POST /api/ai`

**请求体**：

```json
{
  "messages": [
    { "role": "user", "content": "用户输入内容" }
  ],
  "systemPrompt": "系统提示词（可选）"
}
```

**成功响应**：

```json
{
  "content": "AI 生成的文本内容"
}
```

**错误响应**：

```json
{
  "error": "错误信息描述"
}
```

**状态码**：

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 500 | AI 调用失败 |

### 4.2 各模块调用规范

| 模块 | systemPrompt | messages 格式 |
|------|-------------|--------------|
| 文档摘要 | 专业文档处理助手 | `[{ role: "user", content: "请生成摘要：\n\n{input}" }]` |
| 文档关键点 | 专业文档处理助手 | `[{ role: "user", content: "请提取关键点：\n\n{input}" }]` |
| 文档格式化 | 专业文档处理助手 | `[{ role: "user", content: "请格式化整理：\n\n{input}" }]` |
| 会议纪要 | 专业会议助手 | `[{ role: "user", content: "请生成纪要：\n\n{input}" }]` |
| 会议待办 | 专业会议助手 | `[{ role: "user", content: "请提取待办：\n\n{input}" }]` |
| 会议跟进 | 专业会议助手 | `[{ role: "user", content: "请生成跟进：\n\n{input}" }]` |
| 待办排序 | 任务管理助手(JSON) | `[{ role: "user", content: "请排序：\n{todoList}" }]` |
| 任务分解 | 项目管理助手 | `[{ role: "user", content: "请分解：\n{task}" }]` |
| 数据分析 | 数据分析助手 | `[{ role: "user", content: "请分析：\n{dataStr}" }]` |
| 百词斩 | 英语教学助手(JSON) | `[{ role: "user", content: "请生成题目" }]` |

## 6. 安全设计

### 6.1 AI 调用安全

| 措施 | 说明 |
|------|------|
| 后端代理 | z-ai-web-dev-sdk 仅在后端 API Route 中使用，认证信息不暴露给客户端 |
| 输入校验 | 前端校验非空，后端可扩展校验（长度限制、内容过滤） |
| 错误脱敏 | AI 调用失败时返回通用错误信息，不暴露内部细节 |

### 6.2 数据安全

| 措施 | 说明 |
|------|------|
| 本地存储 | 数据仅存储在用户浏览器的 localStorage，不上传服务器 |
| 无用户认证 | v1.0 为单用户模式，无需认证系统 |
| 无敏感数据 | 不收集、不传输用户个人身份信息 |

## 7. 可扩展性设计

### 7.1 模块扩展

新增模块的步骤：
1. 在 `src/components/modules/` 创建新组件
2. 在 Zustand Store 中添加相关状态和 action
3. 在 `page.tsx` 的 `NAV_ITEMS` 数组中添加导航项
4. 在 `ModuleContent` 组件中添加路由分支

### 7.2 AI 能力扩展

新增 AI 功能的步骤：
1. 定义新的 systemPrompt 常量
2. 在模块组件中添加按钮和 handleAction 函数
3. 调用统一的 `fetch('/api/ai')` 接口

### 7.3 数据库扩展

如需从 SQLite 迁移到 PostgreSQL：
1. 修改 `prisma/schema.prisma` 中的 datasource provider
2. 更新 `DATABASE_URL` 环境变量
3. 运行 `bun run db:push`
4. 业务代码无需改动（Prisma Client 抽象了数据库差异）

## 8. 性能优化

### 8.1 前端优化

| 策略 | 实现 |
|------|------|
| 组件懒加载 | 可通过 `dynamic(() => import())` 按需加载模块 |
| 状态持久化 | Zustand persist 减少 API 调用 |
| 动画性能 | 使用 CSS transform/opacity 代替 top/left |
| 列表虚拟化 | 长列表使用 `max-h-96 overflow-y-auto` 限制渲染范围 |

### 8.2 后端优化

| 策略 | 实现 |
|------|------|
| API 路由缓存 | Next.js 自动缓存静态路由 |
| 请求合并 | 所有 AI 调用走统一接口，便于添加缓存/限流 |
| 错误恢复 | AI 调用失败时前端提供 fallback 数据 |
